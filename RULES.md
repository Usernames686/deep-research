# 质量红线与常见陷阱 — deep-research

所有规则按"在哪拦截"标注，违规即降级。

---

## 阶段1 规则

| 规则 | 说明 | 在哪拦截 |
|:----|:-----|:---------|
| **章节约束** | `profiles.json` 是章节数、sections、段落、表格和抓取限额的唯一运行时来源 | 阶段1 `check-outline` |
| **可搜索性** | 每个子问题提供自然语言搜索词；high 问题还需反方词，大纲建议源用于 `site:` 定向查询 | 阶段1 大纲 agent |
| **搜索有界** | 每个问题的候选结果和抓取 URL 上限由 `profiles.json` 控制；禁止无界补强循环 | 阶段2A |
| **对比视角** | 至少 1 个反方观点子问题 | 阶段1 大纲 agent |
| **子节结构完整性** | 大纲 agent 必须为每章定义 sections[]，数量由当前 profile 验收 | 阶段1 `check-outline` |

## 阶段2 规则

| 规则 | 说明 | 在哪拦截 |
|:----|:-----|:---------|
| **Scrapling 为默认抓取工具** | 标准批量抓取优先；仅失败 URL 升级动态浏览器抓取。MCP 未加载时使用同源 `fetch-pending` CLI 兜底 | 阶段2B |
| **抓取可恢复** | 每批结果立即写入 `task2-progress.json`；成功 URL 不得重复抓取，每个 URL 最多标准 + 动态两次尝试 | 阶段2B |
| **结构化事实** | 数据池使用 JSON facts[] 数组格式，非散装文本 | 阶段2 输出 |
| **跨源矛盾标记** | 不同来源矛盾数据在 controversies[] 数组中对齐 | 阶段2 数据池 |
| **来源稀缺** | 失败或无公开数据必须进入 `gaps` 和 manifest 诊断，不得用搜索摘要或虚构事实填补 | 阶段2B |
| **不重复** | 每个新子问题前检查数据池已有内容 | 阶段2 Step 5（数据提取） |
| **来源多样性** | manifest 记录独立域名和逐问题覆盖；不足时标记 data_limited，不阻塞有证据的报告生成 | 阶段2B manifest |
| **数据时序分类** | 每条 fact 必须标注 `data_type`（actual/estimate/forecast）。若该指标的官方发布日在报告生成日之后，自动标记为 estimate 或 forecast | 阶段2 数据池 |

## 章节与装配规范

| 规则 | 说明 | 在哪拦截 |
|:----|:-----|:---------|
| **标题自解释** | 标题不含 C1/ch-01 等内部编号 | 阶段4 QA |
| **子节编号体系分离** | 子节标题必须使用阿拉伯数字 x.x 格式（1.1, 1.2…） | 章节 prompt + `validate-all-chapters` |
| **时间戳硬编码** | 文件名和报告尾时间必须来自运行时系统时间 | 阶段4 装配 |
| **章节 agent 不写最终文件** | 只写临时章节文件，主 agent 统一装配 | 阶段3/4 |
| **章节优先并行** | 有子 agent 能力时同批并行派发；无该能力时按同一文件契约顺序兜底 | 阶段3 |
| **章节验收** | 章节 agent 只写文件，不调用工具；主 agent 使用 outline、mode、lang 批量验证并只重写失败章节 | 阶段3/4 |
| **跨源一致** | 同一数据在不同章节数值一致 | 阶段4 装配检查 |
| **四段式顺序** | 报告结构固定为：标题 → 元数据块 → TOC → 正文与尾部，不可调换 | 阶段4 QA |
| **参考来源行** | 元数据后另起参考来源行，标签与计数格式由语言配置生成 | 阶段4 QA |
| **报告尾部** | 末尾固定包含当前语言的参考来源章节和免责声明 | 阶段4 QA |
| **TOC 标题唯一** | 报告有且仅有一个当前语言对应的 TOC 标题 | 阶段4 QA |
| **乱码零容忍** | data-pool.json 和最终报告不得含替换字符、典型 Mojibake 或连续问号 | Task 2B + 阶段4 QA |
| **预测值措辞限定** | estimate/forecast 事实必须使用预测性措辞，不得当作既定事实 | 阶段3 prompt + 阶段4 QA |
| **纯文本公式** | 报告不得使用 LaTeX/math 语法，全部用纯文本表达 | 阶段3 prompt + 阶段4 QA |
| **可信评估标配** | 参考来源前必须有当前语言的可信评估，由 data-pool + manifest 自动聚合 | 阶段4 装配与 QA |
| **数据类型行必含** | 可信评估必须展示 actual/estimate/forecast 分布 | 阶段4 QA |
| **评估意见动态生成** | 评估意见以统计结果决定基调，使用报告语言做定性判断；不重复逐项数字，不使用固定模板 | 阶段4 LLM 判断 |
| **路径核验** | 最终报告必须保存在 skill 默认目录或用户自定义目录 | 阶段4 QA |
| **编码洁净** | 所有中间文件必须使用 UTF-8 无 BOM，写入后由工具检查 | 阶段1/2/3/4 |
| **清理** | 仅 QA 与索引更新成功后调用带标记校验的 `cleanup-run`；失败运行保留供恢复 | 阶段4 |

## 常见陷阱

| 反模式 | 在哪拦截 |
|:-------|:--------|
| 数值量级错误（亿/billion 差 10x） | Task 2B 事实提取 + 阶段4 QA |
| 完美平滑趋势（虚假插值） | Task 2B 数据提取 + 阶段3 写作 |
| 来源混淆（出货量 vs 零售量） | Task 2B 结构化上下文 |
| 数据不足硬撑行数 | Task 2B gap 诊断 + 阶段4段落约束 |
| **Scrapling 全文被搜索引擎摘要替代** | **阶段2B 严格 data-pool 检查** |
| **内部编号泄露到正文**（如出现 "以下是C1的分析"） | **阶段3 prompt + 阶段4 QA** |
| **自定义脚本替代 dr_tools.py** | **所有阶段优先使用已注册命令** |
| **子节使用一、二、三编号** | **阶段3 prompt + `validate-all-chapters`** |
| **章节 agent 写最终文件**（并行竞态） | **阶段3 临时文件契约** |
| **并行章节数据不一致** | **阶段4 装配一致性检查** |
| 大纲超约束 | 阶段1 硬约束章节数 |
